# Greenlight Review

Datalyz is a technical project focused on the collection, analysis, and visualization of user data in order to better understand profiling, tracking mechanisms, and hyper-personalization online.

The purpose of this folder is to give the jury a clear, structured, and professional overview of the current state of the project, the progress already achieved, the elements demonstrated live, and the roadmap toward final delivery in PGE5.

## Folder structure

```text
greenlight-review/
├── README.md
├── beta-test-plan/
├── track-objectives-report/
├── demo-material/
└── roadmap-pge5/
```

## Deliverables included

### 1. Beta Test Plan

This section defines the features and scenarios currently tested in the project.
It reflects the actual implemented state of Datalyz and serves as the reference for the live demonstration.

### 2. Track Objectives Report

This section presents the objectives associated with our chosen track and provides evidence of progress.
For each objective, the report explains:

- the objective itself;
- why it is relevant to the project and the track;
- what has already been achieved;
- what evidence can be shown;
- what remains to be completed.

### 3. Demo Material

This section contains screenshots, links, and supporting material illustrating working use cases taken from the Beta Test Plan.
Its purpose is to help the jury quickly identify what is already functional and what is demonstrated during the live review.

### 4. Roadmap PGE5

This section provides the roadmap toward final delivery.
It includes the remaining features, technical challenges, identified risks, mitigation strategies, and overall prioritization for the last phase of the project.

## Recommended reading order

To review the project efficiently, the suggested reading order is:

1. **Beta Test Plan**
2. **Track Objectives Report**
3. **Demo Material**
4. **Roadmap PGE5**

This order follows the logic of the Greenlight evaluation and helps connect the current implementation, the objectives, the live demonstration, and the path to completion.

## Current positioning of the project

Datalyz is being evaluated as a **technical track** project because its core value lies in the design, implementation, and validation of a complete technical solution.
The project depends on the robustness of its architecture, the reliability of data collection, the quality of the analysis pipeline, the clarity of the visualization layer, and the security of the overall platform.

## Jury note

All documents in this folder are intended to reflect the **current state of progress** of the project.
The objective is not only to present intentions, but to show concrete progress, working features, technical maturity, and a realistic path toward completion.
